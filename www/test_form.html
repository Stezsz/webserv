webserv - Testes de Funcionalidades HTTP

1. Teste de Formulário (application/x-www-form-urlencoded)

Info: Este formulário usa application/x-www-form-urlencoded. Os dados serão parseados e exibidos na resposta.

2. Teste de Upload de Arquivo (multipart/form-data)

Info: Este formulário usa multipart/form-data para upload de arquivos. O arquivo será salvo no diretório configurado no servidor.

3. Teste de Query String (GET)

Info: Este formulário envia dados via query string (URL). Exemplo: /test?search=webserv+test&category=HTTP&limit=10

4. Teste de Cache Headers

Recarregue esta página várias vezes e observe os headers:

  • Last-Modified: Indica quando o arquivo foi modificado
  • ETag: Identificador único baseado no conteúdo
  • Cache-Control: Controle de cache

Teste com DevTools (F12) → Network → Headers

Info: O servidor deve retornar 304 Not Modified se o arquivo não mudou.

5. Teste de DELETE

Use curl ou fetch para testar o método DELETE:

curl -X DELETE http://localhost:8080/test.txt

# Ou com JavaScript (abra o Console - F12):
fetch('/test.txt', { method: 'DELETE' })
    .then(r => console.log('Status:', r.status))
Info: DELETE deve retornar 204 No Content se bem-sucedido, ou 403 Forbidden se sem permissões.

6. Teste de Chunked Transfer Encoding

Teste com curl enviando dados em chunks:

curl -X POST http://localhost:8080/test \
  -H "Transfer-Encoding: chunked" \
  --data-binary @- << EOF
5
Hello
6
 World
0

EOF
Info: O servidor deve decodificar os chunks automaticamente e processar o body completo.
